Eventos programables JavaScript dentro de una página Web

  1. DOMContentLoaded

    Definición: Se dispara cuando el HTML ha sido completamente cargado y analizado, sin esperar a imágenes o estilos.
    Sirve para: Ejecutar código cuando el DOM ya está disponible.
    Ejemplo: Mostrar un mensaje al cargarse la estructura de la página.

  2. load

    Definición: Se ejecuta cuando la página y todos sus recursos (imágenes, scripts, estilos) han sido cargados completamente.
    Sirve para: Ejecutar código cuando todo el contenido visual esté listo.
    Ejemplo: Mostrar mensaje al terminar de cargar todo.

  3. unload

    Definición: Se dispara cuando el usuario abandona la página (por ejemplo, al cerrar la pestaña o ir a otro enlace).
    Sirve para: Guardar datos o limpiar recursos antes de salir.
    Ejemplo: Mostrar alerta al cerrar la página.

  4. beforeunload

    Definición: Se dispara justo antes de que la página se cierre o se recargue.
    Sirve para: Confirmar si el usuario realmente desea salir o recargar.
    Ejemplo: Mostrar advertencia antes de cerrar la pestaña.

  5. visibilitychange

    Definición: Se activa cuando la pestaña cambia de visibilidad (el usuario cambia de pestaña o vuelve).
    Sirve para: Pausar o reanudar animaciones o videos cuando el usuario deja de ver la página.
    Ejemplo: Mostrar mensaje si la pestaña está oculta o visible.

  6. readystatechange

    Definición: Se activa cuando cambia el estado de carga del documento (`loading`, `interactive`, `complete`).
    Sirve para: Detectar en qué fase de carga se encuentra el documento.
    Ejemplo: Mostrar el estado actual del documento en consola.

  7. fullscreenchange y fullscreenerror

    Definición: `fullscreenchange` se activa cuando un elemento entra o sale del modo pantalla completa. `fullscreenerror` se dispara si ocurre un error al intentar cambiar de modo.
    Sirve para: Controlar el estado del modo pantalla completa.
    Ejemplo: Cambiar a pantalla completa al presionar un botón.

  8. copy, cut, paste

    Definición: Se activan cuando el usuario copia, corta o pega texto en la página.
    Sirve para: Detectar o controlar acciones del portapapeles.
    Ejemplo: Mostrar mensaje cuando el usuario copia o pega algo.

  9. selectionchange

    Definición: Se dispara cuando cambia la selección de texto dentro del documento.
    Sirve para: Detectar qué texto seleccionó el usuario.
    Ejemplo: Mostrar el texto seleccionado en consola.

  10. scroll

    Definición: Se dispara cuando el usuario desplaza la página hacia arriba o abajo.
    Sirve para: Activar animaciones o efectos al hacer scroll.
    Ejemplo: Mostrar la posición de scroll en consola.
